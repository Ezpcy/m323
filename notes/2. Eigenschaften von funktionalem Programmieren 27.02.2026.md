# 2. Eigenschaften von funktionalem Programmieren 27.02.2026

## Referenzielle Transparenz

- Ein Ausdruck (oder ein Funktionsaufruf) ist dann referenziell transparent, wenn du ihn im Code **durch sein Endergebnis ersetzen kannst, ohne dass sich das Verhalten des Programms in irgendeiner Weise ändert.**

**Beispiel**
```scala
def multipliziere(a: Int, b: Int): Int = a * b

// Aufruf:
val ergebnis = multipliziere(3, 4) + multipliziere(3, 4)
```

- Wenn eine ich einen Funktionsaufruf durch seinen Rückgabe Wert ersetzen kann, dann ist die Funktion pur und **referenziell Transparent**

```scala 
val ergebnis = 12 + 12
```

---

## Pure Functions

**Pure Functions** sind Deterministisch, das heisst:
- Das Resultat ist immer das gleiche, egal was als Input gegeben wird.
- **Keine Seiteneffekte**: Die Funktion ändert nichts an der Umgebung.
