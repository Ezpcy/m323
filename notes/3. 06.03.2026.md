## Call by Name

Beim **Call by Name** wird Ausdruck **nicht sofort berechnet**. **Es wird 3 + 4 übergeben**.

```scala
def f(x: => Int): Unit = {
	println(x)
	println(x)
}
f(3 + 4)
```

Jedes Mal wenn `x` benutzt wird, wird 3 + 3 neu berechnet. Berechnung nur wenn nötig.

## Rekursion

Eine Funktion **ruft sich selbst wieder auf**, um ein Problem Schritt für Schritt zu lösen. Wichtig sind immer **zwei Dinge**:
- **Abbruchbedingung (Base Case)**: wann stoppen wir?
- **Rekursiver Schritt:** wie nähern wir und dem Ende?

```scala
def factorial(n: Int): Int =
	if (n == 0) 1
	else n * factorial(n-1)
```

Probleme werden natürlicher, wenn sie rekursiv gelöst werden:
- Ordnerstrukturen
- Bäume
- Graphen
- XML

**Funktionale Programmierung** vermeidet veränderbare Variablen:
- Loops brauchen meist var.
- Rekursion funktioniert mit **immutable Daten**. Das passt zum Stil von **Functional Programming**.
